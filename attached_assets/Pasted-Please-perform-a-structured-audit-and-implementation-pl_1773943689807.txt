Please perform a structured audit and implementation plan for bringing supplier-held available eSIM stock into our Admin Dashboard, starting with eSIM Access, but designed so we can support multiple resellers later (for example eSIMCard as well).

Context:
- We do NOT want to mix this up with the public customer-facing plan catalog.
- This is for internal admin/operations use only.
- The goal is to surface the available eSIMs I already hold in my supplier account(s), such as the ones visible in the eSIM Access “My eSIM” area.
- This should sit under the Admin Dashboard, most likely in the Inventory area, as a new supplier-inventory read model.
- Phase 1 should be read-only. Do not wire it into customer checkout or fulfillment yet.
- We want to build this in a supplier-agnostic way so eSIM Access is first, but eSIMCard and others can be added later.

  - reusable

    - available / unused eSIMs

    - expired eSIMs
    - disabled / revoked eSIMs

- supplier_inventory_items

- status
- sellable / available flag
- data left

- Inventory tab
