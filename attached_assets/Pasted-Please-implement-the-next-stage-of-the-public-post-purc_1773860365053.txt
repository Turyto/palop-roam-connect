Please implement the next stage of the public post-purchase experience: the “Ready to Activate / QR Code / Activation Instructions” order flow, while first auditing the existing codebase to see whether this logic already exists anywhere from the previous application and can be reused.

Main objective:
Define and implement a clear public eSIM activation-state experience inside the Orders / Order Details flow, so users can understand whether their eSIM is still being prepared, ready to activate, already active, expired, or needs support.

Important: before building anything new, first inspect the codebase carefully to identify whether any existing QR code / activation / delivered eSIM / activation-instructions logic already exists from the previous version of the application.

Strategy:
- Reuse existing working logic if it already exists and is still valid
- Do not rebuild QR/activation logic from scratch if the previous app already contains usable code, states, or data mappings
- Preserve backend/order/admin/auth behavior

Core UX goal:
clear status → clear next action → easy activation → support if needed

Critical constraints:

1. Preserve order/backend behavior
- Do not break order retrieval
- Do not break provisioning logic
- Do not break payment status logic
- Do not break admin/order-management behavior
- Do not change backend schemas unless strictly necessary, and if needed, keep changes backward-compatible

2. Preserve auth/admin/session behavior
- Do not break login/authentication
- Preserve Admin access
- Preserve user access/session behavior
- Preserve protected routes and admin dashboards

3. Preserve bilingual behavior
- Reuse the same PT/EN localization system already used on the updated public site
- Do not hardcode public-facing strings outside the translation/i18n structure
- Ensure all new labels/messages/buttons in this feature work correctly in both PT and EN

4. Keep the already-modernized public shell
- Do not regress the current modernized Homepage / Plans / Compatibility / Purchase / Orders shell
- This task is specifically about order activation states, QR visibility, activation guidance, and next actions

PHASE 3 — Implement the “ready to activate” order experience

When an order is in the “ready” state, the user should clearly see:

A. Status block
PT:
- Title: "Pronto para ativação"
- Message: "O teu eSIM está pronto. Podes instalar agora com o QR Code ou com os dados de ativação manual."

EN:
- Title: "Ready to activate"
- Message: "Your eSIM is ready. You can install it now using the QR code or manual activation details."

B. QR / activation section
If QR exists or is available:
- Show QR code
- Show button: PT "Descarregar QR Code" / EN "Download QR code"
- Show button: PT "Copiar dados de ativação" / EN "Copy activation details" (only if manual install data is real and available)
- Show button/link: PT "Ver instruções de ativação" / EN "View activation guide"

If QR is not yet available:
- Do not show placeholder/fake QR
- Show waiting state instead

C. Email reassurance
Show a clear line such as:
PT:
"Também enviámos o teu eSIM e as instruções de ativação por email."
EN:
"We also sent your eSIM and activation instructions by email."

PHASE 4 — Improve the “processing” state

When payment succeeded but eSIM is not ready yet:
Show a clear waiting state.

PT:
Title: "Ativação em preparação"
Message: "Recebemos o teu pagamento. Estamos a preparar o teu eSIM. O QR Code e as instruções de ativação ficarão disponíveis aqui e também serão enviados por email."

EN:
Title: "Activation in progress"
Message: "We received your payment. We are preparing your eSIM. The QR code and activation instructions will appear here and will also be sent by email."

Actions allowed:
- View details
- Talk to support
Optional:
- Refresh status, only if real and implemented

Actions not allowed:
- no fake QR
- no premature download action
- no top-up in this state
