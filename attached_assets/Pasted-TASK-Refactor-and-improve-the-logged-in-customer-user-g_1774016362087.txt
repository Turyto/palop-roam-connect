NEW DASHBOARD STRUCTURE:
Create a simple dashboard with these sections in this order:

A. Dashboard header
- Title: "A minha conta" (or existing i18n equivalent)
- Subtitle with welcome text using customer email/name if available
- No homepage hero copy

B. Current eSIM / plan summary card
This is the top priority card.
Display, when available:
- Plan name
- Status
- Coverage
- Validity / duration
- Purchase date or activation-related date if available
- Main primary CTA depending on status

Supported status labels:
- "Sem plano ativo"
- "Pronto para instalar"
- "Ativo"
- "Expirado"

Map existing backend/provider status into these UX labels without changing backend logic.

CTA behavior by status:
- No active plan -> button: "Comprar eSIM"
- Ready to install -> button: "Ver QR Code" and/or "Instalar eSIM"
- Active -> button: "Ver detalhes"
- Expired -> button: "Comprar novo plano"

Do not invent unsupported actions. If top-up does not exist yet, do not add top-up as a real action.

C. Next step card
Title: "Próximo passo"
Show a contextual instruction block based on state:

State: no plan
- Message: "Ainda não tens um eSIM ativo."
- CTA: "Ver planos" or "Comprar eSIM"

State: ready to install
- Show short step list:
  1. Abre o QR Code
  2. Instala o eSIM no telemóvel
  3. Ativa os dados móveis no destino
  4. Liga o roaming de dados
- CTA: "Ver instruções de instalação"

State: active
- Message: "O teu eSIM está ativo e pronto a usar."
- CTA: "Ver detalhes do eSIM"

State: expired
- Message: "O teu plano expirou."
- CTA: "Comprar novo plano"

D. Order / eSIM details card
Title: "Detalhes do pedido"
Display only meaningful essentials if available:
- Email
- Order number
- Plan name
- Purchase date
- Validity
- Maybe ICCID only in secondary/smaller text if already available
- Keep technical/provider fields hidden unless clearly useful

Add a CTA only if an existing destination/action already exists:
- "Ver detalhes do pedido"
- or "Ver recibo"
Use existing routes/actions if present.

E. Help and support card
Title: "Ajuda"
Include short support-oriented text:
- Problemas com instalação
- Sem ligação à internet
- Questões de compatibilidade

Buttons/links:
- "Contactar suporte"
- "Ver compatibilidade"
- "Ver ajuda" or "Perguntas frequentes"
Only link to routes/pages/actions that already exist.
