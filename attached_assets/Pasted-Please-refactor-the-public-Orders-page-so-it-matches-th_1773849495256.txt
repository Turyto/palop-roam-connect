Please refactor the public Orders page so it matches the updated Homepage, Plans page, Compatibility page, and Purchase/Confirmation flow, while preserving the current working order data, provisioning status logic, and admin/order-management backend behavior.

Main objective:
Modernize the public Orders page so the confirmation CTA “View My Orders” leads to a clean, consistent, user-facing order experience instead of the old legacy orders dashboard.

Current problem:
The funnel is now:
Homepage → Plans → Purchase → Confirmation

This creates a legacy break in the public post-purchase journey.

Desired outcome:
The Orders page should become a modern, simple public account/orders view that helps the user:
1. see what they bought
2. understand its status
3. know whether the eSIM is provisioning or ready
4. access next actions like QR code, activation help, or support

- Do not break login/authentication

- Preserve protected routes and admin dashboards

- Reuse the same PT/EN localization system already used by the updated Homepage, Plans page, Compatibility page, and Purchase page
- Do not hardcode public strings outside the translation/i18n structure

4. Keep the working public route
- Keep or confirm the intended public Orders route
